filemanager.domain.uploads package¶

Provides UploadWorkspace, the organizing concept for this service.

Because UploadWorkspace has many properties and methods, its members are split out into separate classes.

  • BaseWorkspace provides the file index, and foundational methods.

  • FilePathsWorkspace adds methods for working with paths inside the workspace.

  • StoredWorkspace adds integration with a storage adapter.

  • FileMutationsWorkspace adds methods for manipulating individual files.

  • CheckableWorkspace adds slots for checkers and checking strategy, and methods for performing checks. It also adds:

    • SourceTypeMixin, which adds a representation of the submission source upload type.

    • CountableWorkspace, which adds methods for counting files and types of files.

  • UploadWorkspace extends CheckableWorkspace to add an initialization method, and also incorporates:

    • ReadinessWorkspace, which adds semantics around the “readiness” of the workspace for use in a submission.

    • SingleFileWorkspace, which adds the concept of a “single file submission.”

    • LockMixin, which adds support for locking/unlocking the workspace.

    • StatusMixin, which adds the concept of a workspace status.

initialize()[source]¶

Make sure that we have all of the required directories.

This is performed on demand, rather than as a __post_init__ hook, so that we have an opportunity to attach an updated FileIndex after the UploadWorkspace is instantiated but before any system files are created.
